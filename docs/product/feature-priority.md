# Feature Priority

## P0 — Commercial Launch Blockers

These features are required for the first commercial release. The platform cannot launch without every P0 item.

| Feature | Rationale |
|---------|-----------|
| Accounts | Users must register and log in to access paid content and track progress |
| Onboarding | New users must complete setup, target selection and device check before using the platform |
| Courses | Structured lessons are the core learning delivery mechanism |
| Question engine | All task types require a reusable question delivery, response collection and scoring system |
| All current task interfaces | Every PTE Academic task type must be supported for students to practise effectively |
| Speaking recording | Speaking tasks are a major PTE component; the platform must capture and process spoken responses |
| Writing autosave | Written responses must not be lost due to navigation, refresh or interruption |
| Objective scoring | Multiple-choice, fill-in-the-blanks and similar tasks require deterministic scoring |
| Estimated speaking and writing evaluation | AI-based scoring for speaking and writing is a key value proposition |
| Diagnostic test | Baseline assessment is required for personalised study plans |
| Study plan | Personalised daily recommendations drive student engagement and progress |
| Mock exams | Full-length simulations are essential for test readiness |
| Reports | Students must see their scores, progress and improvement areas |
| Admin content management | Content writers and reviewers need tools to create, review and publish questions |
| Payments | Subscription and practice pack purchases must be processed securely |
| Recovery | Student work must survive interruption, session expiry and system failures |
| Monitoring | Platform health, failed jobs and errors must be visible to operations |

## P1 — Launch-Quality Enhancements

These features significantly improve the student experience and competitive positioning. They should be delivered as early as possible after the P0 baseline is stable.

| Feature | Rationale |
|---------|-----------|
| Teacher review | Human expert feedback adds credibility and differentiation from automated-only platforms |
| Mistake notebook | Structured error tracking improves learning outcomes |
| Vocabulary notebook | Vocabulary building supports overall English improvement |
| Spaced repetition | Evidence-based review scheduling increases long-term retention |
| Coupons | Promotional pricing supports marketing and acquisition |
| Weekly reports | Regular progress emails drive re-engagement |
| Advanced analytics | Detailed performance insights help students target weak areas |

## P2 — Post-Launch

These features expand the platform reach and revenue potential after the core product is established.

| Feature | Rationale |
|---------|-----------|
| Native mobile apps | Deeper device integration and offline capability |
| Live classes | Synchronous instruction adds a premium offering |
| Referral system | User-driven acquisition reduces customer acquisition cost |
| Affiliates | Partner-driven distribution expands market reach |
| School accounts | Institutional sales open a new revenue channel |
| White-label platform | Third-party resellers extend the platform's addressable market |
| Public community | User-generated content and peer support increase engagement and retention |
